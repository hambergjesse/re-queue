Dual Detection Technology

RE:QUEUE uses two complementary methods to detect the "Accept" button:

  • MutationObserver Monitors DOM changes to detect the button the moment it appears
  • Polling Detection Regularly checks for the button as a fallback mechanism
  • Adaptive Timing Adjusts polling frequency based on queue status for optimal performance 
// Detection snippet
const observer = new MutationObserver(() => {
  const acceptButton = document.querySelector('.queue-accept');
  if (acceptButton) {
    acceptButton.click();
    showNotification('Match accepted!');
  }
});

// Start observing
observer.observe(document.body, { 
  childList: true,
  subtree: true 
});

Frequently Asked Questions

What is RE:QUEUE?

RE:QUEUE is a browser extension designed specifically for Renown.gg. It automatically accepts match queues even when you're not actively on the Renown tab, ensuring you never miss a match.

How does RE:QUEUE work?

RE:QUEUE uses advanced detection methods to identify when a match is ready on Renown.gg. It then automatically clicks the "Accept" button for you, even if you're browsing in a different tab or window.

Is RE:QUEUE safe to use?

Yes, RE:QUEUE is completely safe. It's open-source, so you can review the code on our GitHub repository. The extension only operates on Renown.gg and doesn't collect any personal data.

Which browsers are supported?

RE:QUEUE works with all major browsers including Chrome, Firefox, Edge, and Opera. Simply install the extension from your browser's extension store.

Can I turn off RE:QUEUE temporarily?

Yes, you can easily toggle RE:QUEUE on and off using the control panel in the extension. This gives you full control over when the auto-accept feature is active.
